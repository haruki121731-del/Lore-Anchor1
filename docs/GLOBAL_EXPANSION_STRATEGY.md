> **Vision:** 世界中のクリエイターをAIによる無断学習から守り、権利を証明し、収益化へ繋げる次世代の著作権管理インフラを構築する
> 
> **Mission:** 10,000,000人のクリエイターを保護する

## 1. 現状分析

### 1.1 プロダクトの現在地

| 項目 | 状態 |
|------|------|
| **コア技術** | ✅ Mist v2 (AI学習妨害) + PixelSeal (透かし) + C2PA (来歴証明) |
| **Webアプリ** | ✅ Next.js + Vercel (稼働中) |
| **バックエンド** | ✅ FastAPI + Railway (稼働中) |
| **GPU Worker** | 🟡 SaladCloud (停止中、コスト削減のため) |
| **マーケティング自動化** | 🟡 一部故障中 |
| **ウェイトリスト** | ⚠️ 1名のみ |
| **月間コスト** | ~$5 (Railway) |

### 1.2 解決する課題の大きさ

```
【世界のクリエイター市場】
├─ イラストレーター/デジタルアーティスト: 約5,000万人
├─ 写真家: 約1,500万人  
├─ 漫画家/同人作家: 約300万人
├─ デザイナー: 約1,000万人
└─ AI学習への懸念を持つクリエイター: 約60% (推定)

→ ターゲット市場: 約4,700万人
```

### 1.3 競合分析

| 競合 | 強み | 弱み | Lore-Anchorの差別化 |
|------|------|------|---------------------|
| **Glaze** | 無料、学術的信用 | ローカル処理のみ、時間がかかる、透かしなし | クラウド処理で高速、透かし+C2PA付き |
| **Nightshade** | 攻撃的（poisoning） | 設定が難しい、ローカルのみ | 使いやすさ、追加の証明機能 |
| **Kudurru** | 監視機能 | 防御機能が弱い | 防御+証明の統合 |
| **Spawning** | opt-out登録 | 技術的防御なし | 技術的防御+法的証明 |

## 6. 法的・倫理的考慮事項

### 6.1 コンプライアンス

```
┌────────────────────────────────────────────────────────────┐
│ GDPR (EU)                                                  │
│ ├─ 画像処理の同意取得                                      │
│ ├─ データポータビリティ対応                                │
│ └─ 忘れられる権利の実装                                    │
├────────────────────────────────────────────────────────────┤
│ CCPA (California)                                          │
│ ├─ プライバシーポリシー明記                                │
│ └─ オプトアウト対応                                        │
├────────────────────────────────────────────────────────────┤
│ AI規制                                                    │
│ ├─ EU AI Act 対応準備                                     │
│ └─ 透明性の確保（C2PA標準準拠）                            │
└────────────────────────────────────────────────────────────┘
```

### 6.2 倫理ガイドライン

1. **透明性:** 処理内容を明確に説明
2. **可逆性:** オリジナル画像は常に保持
3. **ユーザー所有:** 処理済み画像の権利はユーザーに帰属
4. **非攻撃的アプローチ:** 「poisoning」ではなく「protection」
