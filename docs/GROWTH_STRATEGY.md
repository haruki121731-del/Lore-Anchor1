## 3. ゼロコスト成長戦略

### Phase 0: 修復（今日）— コスト: $0
- [ ] マーケティング自動化の故障修復（pip cache エラー）
- [ ] ウェイトリストURL統一（waitinglist-lovat.vercel.app）
- [ ] MEMORY.md のURL更新

### Phase 1: 無料ユーザー獲得エンジン（1週間目）— コスト: $0
1. **Twitter/X 自動化の活性化**
   - 毎日1-2ツイート自動投稿（教育コンテンツ）
   - ハッシュタグ: #AI学習禁止 #イラスト保護 #LoreAnchor #C2PA
   - 内容パターン:
     - 「知ってた？あなたのイラストは今この瞬間もAIに学習されている可能性があります」
     - 「GlazeとLore-Anchorの違い」比較画像
     - 「C2PA署名とは？Adobe/Googleが推進する新標準」
     - クリエイターの不安に寄り添う共感ツイート

2. **日本語SEOコンテンツ**
   - Note.com に記事投稿（無料）:
     - 「イラストレーターが今すぐやるべきAI学習対策5選」
     - 「Glaze vs Nightshade vs Lore-Anchor 徹底比較」
     - 「C2PA署名でAI無断学習に法的に対抗する方法」

3. **Pixiv コミュニティ**
   - プロフィールにLore-Anchor リンク
   - 保護前後の比較画像を投稿

### Phase 2: CPU無料枠の実装（2週間目）— コスト: $0追加
1. **Railway上でCPUワーカーを起動**
   - `Dockerfile.cpu` を使用（既存）
   - Mist v2 の `freq` モード（NumPy DCT、GPU不要）
   - PixelSeal（DWT、GPU不要）
   - C2PA署名（CPU で十分）
   - 処理時間: ~30-60秒/枚（GPU: ~5秒）

2. **月5枚の無料枠制限**
   - Supabase で `images` テーブルの月間カウント
   - 超過時は「Proにアップグレード」CTA表示

### Phase 3: 決済導入（3週間目）— コスト: $0（Stripe手数料のみ）
1. **Stripe Checkout**
   - 月額¥980のサブスクリプション
   - 年額¥9,800の一括払いオプション
   - Webhook → Supabase の `users` テーブルに `plan` カラム追加

2. **プライシングページ**
   - メインアプリのランディングページに料金表追加
   - Free vs Pro の機能比較テーブル

### Phase 4: GPU オンデマンド（収益後）— コスト: 従量課金
1. **SaladCloud scale-to-zero**
   - Pro ユーザーのタスクがキューに入った時だけ起動
   - API からSaladCloud Container Group を `start` / `stop`
   - 15分間アイドルなら自動停止

2. **コスト管理**
   - 1ユーザーあたりの GPU 使用量追跡
   - 月100枚上限で Pro のコストを予測可能に

## 結論

**「お金をかける前にお金が入る」を実現するフロー:**

1. Twitter自動化 + Note.com記事 → **無料で認知拡大**
2. CPU無料枠 → **$0で価値を体感させる**
3. Free → Pro 転換 → **月¥980の収益**
4. Pro ユーザー20名(MRR ~¥20,000) → **GPU運用費を回収**
5. GPU稼働 → **Pro体験向上** → **さらなる転換**

**最も重要な原則: GPU を起動するのは Pro ユーザーの MRR が GPU コストを超えてから。**
